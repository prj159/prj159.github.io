使用git

发表于2026-03-09|更新于2026-03-11
|浏览量:

笔者遇到了一些较为不可思议之事。有人在做项目之时，每做完一份就会把那个版本的文件传到QQ的群文件中，美名其曰保留记录。笔者OS：？？？

事实上，这个行为其实就是简化版的git，但是就是有人不会git。这里笔者只好说，懂git的人可以退出，不懂git的客官请继续看。

下载并安装git

下载是到网站(https://git-scm.com/install/windows)中下载安装包。里面涵盖了Windows、Mac、Linux三大操作系统的安装程序，点进去看就可以看到安装包下载或者教你如何命令行下载。跟着做就可以成功下载并安装git。

如果你想看有没有装好，直接将这个代码敲进你的终端中。

如果可以弹出版本号就是下载完成，如果没有弹出那可能需要在知乎、CSDN等网站找解决方法了。

随后需要配置你的git。注意！你的名字一定要用英文！！！你的邮箱最好用你注册GitHub或者Gitee的邮箱账号。

使用git

如今的大多数人看到命令行就发愁，这里推荐使用GitHub desktop的软件，这个是图形化软件，可以一目了然地学会如何操作git。

首先肯定是要找到GitHub desktop这个软件，安装包在网站(https://desktop.github.com/download/)中下载即可。

使用该软件之前需要使用GitHub账号登陆。接下来的操作笔者会用Mac OS来展示，Windows用户一样是可以跟着操作的。一般Mac OS的菜单栏都是在屏幕的左上角，Windows即是对应软件窗口的左上角。

个人工作流

我们先以个人工作流为例。为了可以获取文件数据，第一步一定是要点击左上角的File，随后会弹出三个选项，分别是：

New repository（新建仓库，当你的GitHub和本地电脑中都没有该项目的时候点击，这样会在你的GitHub和本地电脑都生成项目）

add local repository（添加本地仓库，当你的GitHub中没有该项目，但是本地电脑有该项目的时候点击，这样会把本地电脑的项目上传到GitHub中）

clone repository（克隆仓库，当本地电脑没有项目，但GitHub中有项目的时候点击，这样会把GitHub的项目下载到你的电脑中）

假设我们在GitHub和本地电脑中都没有项目，我们点击New repository。随后会出现该界面

Name（你的仓库名字）

Description（对你仓库的描述）

Local path（把该项目存在你的电脑的哪个地方）

Initialize this repository with a README（默认会生成一个README.md文件，这个文件一般是开发者写给使用者的内容，相当于说明书）

Git ignore（忽略上传哪类文件）

License（许可）

值得注意的是，这个恢复并不是对整个文件的恢复，而是对你改的某几行代码的恢复。

团队工作流

接下来是团队合作工作流。团队合作工作流必然会比个人工作流要复杂一些。

如果你是参与者，你可以让主持人invite你随后就可以clone你们团队的项目到你的电脑上。此时克隆需要在GitHub desktop点击File，随后点击clone repository，就会出现这个界面。在该界面中选择你想要克隆的仓库，然后更改你想要存放的位置，这样就成功地将团队的项目给存到你的电脑中了。

团队工作流经常需要用到的是分支branch和代码冲突。

分支branch是为了给代码加入新功能，但是害怕加新功能之后会改崩最后挂掉的这种情况做准备。我们还是用之前的hello仓库试试水，要新建一个branch，首先要点击current branch，随后点击new branch，给这个新的分支命名，笔者将该分支的命名为test，确认后，现在我们就处在test分支里，如果想要回去main分支，就还是点击current branch再点击main，这样就回到main分支中。

我们在test分支里面将main.py改成图中这样，随后上传，这样就完成了分支的新建和上传。在GitHub中也可以看见你新建的分支。

新功能肯定是要合并到主分支的，在你认为代码测试得差不多时，就可以合并到main分支了。首先将你的current branch调到main，随后要点一下current branch，再点击choose a branch to merge into main，选择你想将哪个分支合并到主分支，笔者这里选择test，最后点击create a merge commit。这样就成功地把test分支的内容合并到主分支。

上图笔者截错了，大家看到操作和图片展示不一样不要慌张

代码冲突出现在你和你的同事在同时改同样的一份文件，然后你的同事先提交了他的代码，随后你也想要提交你的代码，因为他改的部分代码和你改的部分代码因为git没有这么高级导致无法合并，这样GitHub desktop就会报错，说有冲突，这样就需要人去解决这部分的冲突。

笔者这里模拟一下这个冲突，笔者还是在hello仓库里在main分支里新建一个test.txt文件，在里面写着“这是基础版本”。随后在main分支分出一个新的分支test2，将test2分支的test.txt文件的“这是基础版本”改为“这是test2的版本”。再去main分支的test.txt文件的“这是基础版本”改为“这是main的版本”。把test2分支合并到main分支，这样GitHub desktop就会说有冲突。

我们点击open in Visual Studio Code，然后就在VSC中看到因为哪些地方发生了冲突，为了方便观察，我们可以选择比较更改，这样界面就会变成两个分支有改动的文本和原本没有改动的文本。此时笔者选择“这是main的版本”的更改，再点击完成合并，这样冲突就可以解除了。最后再回到GitHub desktop就可以将你的代码上传到GitHub了。

此时笔者选择“这是main的版本”的更改，再点击完成合并，这样冲突就可以解除了。最后再回到GitHub desktop就可以将你的代码上传到GitHub了。

需要注意的是，在团队合作中要经常地点击Fetch origin，以免落掉同事修改的代码。

最后如果想要在GitHub删除hello这个仓库，可以在GitHub网站找到你的hello这个仓库点进去，随后在上方的菜单栏中找到settings，划到最下面，点击delete this repository，跟着系统提示操作，就可以把这个仓库在GitHub中删了。本地的直接移到垃圾桶就删了。
